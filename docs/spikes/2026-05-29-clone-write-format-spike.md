# Clone-on-write file-format manipulation spike (Unit 0)

Date: 2026-05-30
Status: converged (xlsx 100% / dyr 88% of editable params — well above the 80% gate)

## Goal

For each case-file format the substrate ingests (`.xlsx` ANDES-native,
`.dyr` PSS/E dynamics, `.raw` PSS/E steady-state), determine the safe in-place
edit primitive for the R15 dynamic-controller param classes, and produce a
machine-readable `(model, param) → edit_op` index that Unit 21's CloneManager
writer ingests directly.

## Sources (all authoritative, no invention)

- **ANDES param names per model:** `andes_app.core.wrapper._PARAMS_BY_MODEL`
  (the Unit-15 whitelist, itself introspected from the live ANDES model
  classes). `ParamMeta.kind == 'number'` selects the editable scalar params;
  `idx`/`name` and reference params (`syn`, `avr`, `reg`, `ree`, `bus`, `gen`,
  …) bind identity/topology and are immutable.
- **`.dyr` field order + param map:** `andes/io/psse-dyr.yaml`. Per model it
  gives `inputs` (the ordered PSS/E record fields) and `outputs`
  (ANDES-param → PSS/E-field, sometimes with a `; lambda …` transform).
- **`.xlsx` layout:** verified by `openpyxl` against `kundur_full.xlsx` — one
  sheet per model class, header row = ANDES param names, one row per device.

The index is generated by `scripts/gen_clone_write_index.py`
(`PYTHONPATH=server/src python scripts/gen_clone_write_index.py`); its output is
`docs/spikes/2026-05-29-clone-write-index.json`. The drift guard
`server/tests/unit/spike/test_clone_write_index.py` re-validates it against live
ANDES on every test run.

## Per-format edit primitives

### `.xlsx` (ANDES-native) — primary path

Uniform and total. For `(model, param, idx, value)`:

1. Open the workbook; select sheet `model`.
2. Read header row; find the `idx_column` (`"idx"`) and the `param` column.
3. Find the row whose idx-cell equals `idx`; set its `param` cell to `value`.

Every editable param is `.xlsx`-editable because the ANDES exporter writes a
column per param. `kundur_full.xlsx`, `ieee39.xlsx` use this format.

### `.dyr` (PSS/E dynamics)

A record is `BUS 'MODEL' ID f0 f1 … /` and may span multiple physical lines;
fields follow the model's `inputs` order. For `(model, param)`:

1. `outputs[param]` gives the PSS/E field name. If it is absent, or carries a
   `;` transform (the file stores a *different* quantity, e.g. `GENROU.M = 2·H`
   where the file holds `H`), the param is **not** directly `.dyr`-editable.
2. `field_index = inputs.index(field)`.
3. Locate the record matching `(BUS, MODEL, ID)`; replace the field at
   `field_index`. **File-token offset:** the file inserts the quoted model name
   between `BUS` and `ID`, so the file token for `inputs[k]` (k ≥ 1) is at file
   index `k + 1` (`inputs[0]=BUS` is token 0).

> **Critical writer requirement (finding):** the writer MUST edit the target
> token *in place* — preserving every other token and all sibling records.
> A naive "tokenize → re-join the whole record" collapse drops sibling records
> when two devices of the same model exist (observed: editing the first of two
> `ST2CUT` records dropped it, leaving only the second). Unit 21's `dyr_writer`
> should locate the target field by walking tokens with their source character
> offsets and splice only that token's span, leaving the rest of the file
> byte-identical. Integration tests must load a case with **two devices of the
> same model** and assert both survive with only the target edited.

### `.raw` (PSS/E steady-state)

Holds **no** dynamic-controller params (those live in the paired `.dyr`).
Controller edits never route to `.raw`. Static-topology edits (bus Vn, line
impedance) are already covered by the existing `PUT /elements/{model}/{idx}`
route, so the clone `.raw` writer raises `CloneEditError` with
`recovery.ui_hint: 'use-edit-element'` if ever invoked for a controller param.

## Coverage (18 whitelisted controller models, 294 editable params)

- **`.xlsx`: 294 / 294 (100%)** — total.
- **`.dyr`: 259 / 294 (88%)** — exceeds the 80% gate.

`.dyr` gaps are all expected and route to `.xlsx`-only (or save-as-only for a
`.raw`+`.dyr`-loaded case):

| Model    | dyr-editable | Why the rest are xlsx-only |
|----------|--------------|----------------------------|
| PMU      | 0 / 2        | not a PSS/E model (ANDES measurement) |
| REGCP1   | 0 / 18       | ANDES extension of REGCA1; not in psse-dyr.yaml |
| TimeSeries | 0 / 1      | not a PSS/E model |
| TGOV1 / GAST / HYGOV / IEEEG1 / IEESGO | all but 2 each | `Tn`, `wref0` are ANDES-added params absent from PSS/E |
| REGCA1   | 15 / 18      | `Sn`, `gammap`, `gammaq` are ANDES-derived |

All other models (EXST1, ESST1A, ESDC2A, IEEEX1, IEEEST, ST2CUT, REECA1,
REPCA1, SEXS) are 100% `.dyr`-editable.

## Verification (round-trip)

Performed via a one-off harness (not committed; results below):

- **`.xlsx` exact round-trip — PASS.** Edited `TGOV1.T1[idx 1] = 0.6` in a copy
  of `kundur_full.xlsx`, reloaded with `andes.load(..., setup=True)`, observed
  `ss.TGOV1.T1.v[0] == 0.6`.
- **`.dyr` field-mapping — CONFIRMED.** For `ieee14.dyr`'s `ST2CUT_2`, the live
  values match the file tokens at the index's computed `field_index`
  (`T3=30.0`→token 11, `T5=0.23`→token 13, `T7=0.23`→token 15,
  `LSMAX=0.06`→token 19). The end-to-end `.dyr` *edit* round-trip is deferred to
  Unit 21's writer (which implements the in-place splice above); the spike
  harness's whole-record replace is insufficient by design.

### Note on per-unit normalisation

Some params are per-unit-normalised to the system base at `setup()` (e.g. the
governor droop `R`: file `0.0777` → `R.v = 0.0777 · Sb/Sn`). The **file** holds
the device-base value, which is what the writer edits and what the inspector
should display/round-trip; the live `*.v` array is the normalised value. Unit 21
+ Unit 23 (Modified-from-Original diff) compare **file values**, not `*.v`.

## Fallback policy

Per the execution note: any `(model, param)` not resolvable to a format edit op
defaults to **save-as-only** for that format — the inspector renders the param
read-only when the loaded case is in that format, but the user can still edit
the file externally and re-load, and save-as still works. With 100% `.xlsx`
coverage, the only read-only inspector cases are dynamic params of a
`.raw`+`.dyr` case whose param is `.dyr`-gap (the ANDES-added `Tn`/`wref0` and
the non-PSS/E models above).
